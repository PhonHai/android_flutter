## 四、核心架构洞察（朋友透露 + 推演）

### IoT 应用采用「原生 + Flutter 混合架构 (Add-to-App)」

**不是纯 Flutter App**，而是分三层演进：

```
阶段1: 纯原生 Android (Kotlin)
 → 所有模块都是原生 Kotlin 写的

阶段2: 接入 Flutter 模块
 → 核心模块（文件管理、下载引擎、NAS连接）继续用原生 Kotlin
 → 新模块（影视中心、设备发现、分享）用 Flutter 写，Android+iOS 复用
 → MethodChannel 是原生和 Flutter 之间的通信桥梁

阶段3: 鸿蒙适配
 → 把 Android 原生壳换成鸿蒙壳（ArkTS 重写 NAS连接、下载引擎等）
 → Flutter 模块完全不动，同一套 Widget 继续跑
 → Platform Channel 从 Flutter↔Android 改为 Flutter↔鸿蒙
```

**一句话总结**: Flutter 模块是跨端资产，壳按平台换，Flutter 不动。

### IoT 应用技术栈推演

| 层级 | 可能的技术选型 | 原因 |
|------|---------------|------|
| 跨端框架 | Flutter | 需要同时跑 Android+iOS+TV+桌面 |
| 鸿蒙适配 | Flutter × 鸿蒙 | 跨端基础上适配鸿蒙 |
| 状态管理 | Riverpod / Provider | Flutter 生态主流 |
| 网络层 | dio + Retrofit-like | NAS 需要大量 HTTP/WebSocket |
| 本地存储 | sqflite / Hive | 缓存文件列表、用户设置 |
| 原生交互 | Platform Channel | 调用原生文件管理、下载能力 |
| 鸿蒙扩展 | 分布式数据管理 | 跨设备访问 NAS 文件 |

## 五、学习策略（3周计划）

### 优先级

| 优先级 | 学什么 | 为什么 |
|-------|--------|--------|
| **P0** | Flutter + MethodChannel | 日常工作最多，写 Flutter 模块 + 和原生通信 |
| **P0** | 原生 Kotlin 壳维护 | 8年经验已覆盖，需理解 Add-to-App 集成方式 |
| **P1** | 鸿蒙壳替换原理 | 理解 UIAbility + ArkTS 怎么替代 Activity + Kotlin |
| **P2** | Flutter × 鸿蒙适配流程 | 理解 flutter build harmonyos 和鸿蒙 Platform Channel |

### Week 1: Flutter 番茄钟 MVP

- Dart 语言（≈Kotlin，2-3天上手）
- Flutter Widget 树（≈Compose Composable）
- Riverpod 状态管理（≈ViewModel + StateFlow）
- CustomPainter 画圆环倒计时
- sqflite 存历史记录
- Flutter Local Notification

### Week 2: 鸿蒙原生基础

| 概念 | 鸿蒙 | 对应 Android | 深度要求 |
|------|------|-------------|---------|
| 应用入口 | UIAbility | Activity | 必须会 |
| 页面开发 | ArkUI (声明式) | Compose | 能写简单页面 |
| 数据存储 | relationalStore | Room | 知道 API 差异 |
| 状态管理 | @State/@Link | ViewModel+LiveData | 概念映射 |
| 应用模型 | Stage 模型 | Application | 理解就行 |

### Week 3: Flutter × 鸿蒙适配

- Flutter 3.28+ 直接创建鸿蒙平台项目
- Platform Channel 通信（Flutter 调鸿蒙原生 API）
- 响应式多设备布局（LayoutBuilder）
- 嵌入原生 ArkTS 控件（PlatformView）

## 六、番茄钟项目方案（混合架构）

### 项目目标
用混合架构做番茄钟，项目架构直接对标IoT 应用真实架构，面试时可以讲。

### 项目结构

```
PomodoroApp/
├── native-android/ # 原生 Android 壳
│ ├── app/
│ │ ├── MainActivity # 原生首页，导航入口
│ │ ├── PomodoroListActivity # 原生写的番茄钟列表
│ │ └── FlutterContainerActivity # 容纳 Flutter 的 Activity
│ └── build.gradle
│
├── flutter_module/ # Flutter 模块 (作为 AAR 被原生引用)
│ ├── lib/
│ │ ├── main.dart # Flutter 番茄钟计时页面
│ │ ├── timer_widget.dart
│ │ └── channels.dart # MethodChannel 定义
│ └── pubspec.yaml
│
└── harmonyos-shell/ # 鸿蒙壳 (Week 3 再做)
 ├── entry/
 │ ├── UIAbility
 │ └── pages/
```

### 核心学习链路
1. 原生 Kotlin 写壳 + 导航（8年经验，秒过）
2. Flutter 模块写番茄钟计时页面（学 Flutter 核心）
3. MethodChannel 让 Flutter 调原生「保存历史记录」接口（学混合通信）
4. Flutter 编译成 AAR，原生壳引用（学集成流程）

### 面试话术
> 「我做了一个原生 + Flutter 混合架构的番茄钟 App，Flutter 模块通过 MethodChannel 和原生通信，编译为 AAR 集成到原生壳中，同时适配了鸿蒙端。」
